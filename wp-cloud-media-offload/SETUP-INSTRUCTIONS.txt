================================================================================
WP CLOUD MEDIA OFFLOAD - SETUP INSTRUCTIONS
================================================================================

Thank you for choosing WP Cloud Media Offload!

This file contains essential setup instructions to get your plugin running.

================================================================================
STEP 1: INSTALL DEPENDENCIES
================================================================================

The plugin requires AWS SDK for PHP. Install it using Composer:

1. Open terminal/command prompt
2. Navigate to the plugin directory:
   
   cd wp-content/plugins/wp-cloud-media-offload

3. Run Composer install:
   
   composer install

   If you don't have Composer installed:
   - Visit: https://getcomposer.org/download/
   - Follow installation instructions for your OS
   - Then run: composer install

4. Verify installation:
   - Check that /vendor/ folder exists
   - Check that /vendor/aws/ folder exists

================================================================================
STEP 2: ACTIVATE PLUGIN
================================================================================

1. Log in to WordPress admin
2. Go to Plugins > Installed Plugins
3. Find "WP Cloud Media Offload"
4. Click "Activate"

================================================================================
STEP 3: ACTIVATE LICENSE
================================================================================

1. Go to Cloud Media > License
2. Enter your license key
3. Click "Activate License"
4. Wait for confirmation

Don't have a license key?
- Visit: yoursite.com/pricing
- Choose your plan
- Complete purchase
- License key will be emailed to you

================================================================================
STEP 4: CONFIGURE AWS
================================================================================

Before configuring the plugin, you need:

A. AWS S3 Bucket
   - Log in to AWS Console
   - Go to S3 service
   - Create a new bucket
   - Uncheck "Block all public access"
   - Note the bucket name and region

B. IAM User Credentials
   - Go to IAM service
   - Create new user with programmatic access
   - Attach S3 permissions policy
   - Save Access Key ID and Secret Access Key

For detailed AWS setup instructions, see:
- docs/AWS-SETUP-GUIDE.md
- Or visit: yoursite.com/docs/aws-setup

================================================================================
STEP 5: CONFIGURE PLUGIN
================================================================================

1. Go to Cloud Media > Settings
2. Enter AWS credentials:
   - Access Key ID
   - Secret Access Key
   - Region (e.g., us-east-1)
   - Bucket Name
3. Click "Test Connection"
4. If successful, click "Save Settings"

Optional - CloudFront CDN:
1. Create CloudFront distribution in AWS
2. Point it to your S3 bucket
3. In plugin settings:
   - Enable "CloudFront CDN"
   - Enter CloudFront domain
4. Save settings

================================================================================
STEP 6: START UPLOADING
================================================================================

Option A - Auto Upload (New Files):
1. Settings > Enable "Auto Upload New Media"
2. Upload images normally in WordPress
3. Files automatically go to S3

Option B - Bulk Upload (Existing Files):
1. Go to Cloud Media > Bulk Upload
2. Click "Start Bulk Upload"
3. Keep browser tab open
4. Monitor progress

================================================================================
TROUBLESHOOTING
================================================================================

Connection Test Fails:
- Verify AWS credentials are correct
- Check IAM user has S3 permissions
- Confirm bucket name and region match
- Check server can connect to AWS

Images Not Loading:
- Verify bucket has public read access
- Check files uploaded to S3
- Verify CloudFront domain if enabled
- Check browser console for errors

Bulk Upload Stops:
- Refresh page and restart
- Already uploaded files are skipped
- Check PHP error logs
- Increase PHP max_execution_time

Composer Install Fails:
- Verify Composer is installed correctly
- Check PHP version (7.4+ required)
- Try: composer install --no-dev
- Check server has internet access

SYSTEM REQUIREMENTS
================================================================================

Minimum:
- WordPress 5.8+
- PHP 7.4+
- MySQL 5.6+
- 128 MB PHP memory limit
- cURL enabled
- OpenSSL enabled

Recommended for Large Libraries:
- PHP 8.0+
- 256 MB+ PHP memory limit
- 300+ seconds max_execution_time
- Fast internet connection

================================================================================
TIPS FOR LARGE LIBRARIES
================================================================================

For 25,000 Images:
- Time: 2-4 hours
- Storage: ~2.5 GB (avg 100KB/image)
- Cost: ~$9/month

For 250,000 Images:
- Time: 20-40 hours
- Storage: ~25 GB (avg 100KB/image)
- Cost: ~$88/month

Best Practices:
- Run bulk upload during off-peak hours
- Keep browser tab open during upload
- Monitor server resources
- Test with 1,000 images first
- Backup before major changes
